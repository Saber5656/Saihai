# Directory Path Aliases

This tracked file is a public alias catalog. Do not put concrete machine paths,
usernames, private vault names, or environment-specific examples here.

Set actual local values in the ignored `directory-path.env`. Saihai loads that
file through `directory_paths.py`; process environment values override matching
entries for a single invocation.

| Alias | Local value | Purpose |
|---|---|---|
| `SAIHAI_ROOT` | set locally | Saihai source repository |
| `SAHAI_ROOT` | set locally | Saihai source repository (legacy misspelling alias, kept for compatibility) |
| `AGENT_TEAMS_VIEWER_ROOT` | set locally | Sahai source repository (legacy Agent-Teams-Viewer alias, kept for compatibility) |
| `AGENTS_VAULT_ROOT` | set locally | shared organization vault |
| `USER_VAULT_ROOT` | set locally | personal Obsidian vault |
| `SKILLS_REPO_ROOT` | set locally | skills source repository |
| `SKILLS_ROOT` | set locally | skill definitions |
| `DOTFILES_ROOT` | set locally | local agent/runtime configuration |
| `DEV_ROOT` | set locally | local source repositories |
| `DEV_WORKTREES_ROOT` | set locally | task worktrees |
| `TASK_WORKTREE_ROOT` | set locally | temporary task worktrees |
| `SAIHAI_ORCH_STATE_ROOT` | optional | orchestrator state override |
| `SAIHAI_ITB_STATE_ROOTS` | optional | ITB state-root path list |
| `SENSITIVE_ACCESS_GUARD_STATE_ROOT` | optional | sensitive-access guard state |

Legacy aliases remain read-compatible: `SAHAI_ROOT`,
`AGENT_TEAMS_VIEWER_ROOT`, `YASU_VAULT_ROOT`,
`SKILLS_REPO_SKILLS_ROOT`, `DEV_REPO_ROOT`,
`SAHAI_ORCH_STATE_ROOT`, and `SAHAI_ITB_STATE_ROOTS`.
