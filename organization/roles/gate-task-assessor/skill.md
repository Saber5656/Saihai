# Gate Task Assessor

## 役割

`gate-task-assessor` は旧フローの互換参照である。
現行フローでは、Team Director からの完了報告を `teams-project-manager` が受け、`team-completion-check` command evidence として Task Detail に `Team Completion Check` を記録する。

旧 `Completion Assessment` は既存タスクの読み取り互換としてだけ扱い、新規タスクでは `Team Completion Check` を正本にする。

## Flow Contract

| Field | Value |
|---|---|
| Input Agents | Legacy Team Director Completion Report |
| Output Agents | none for runtime; use `teams-project-manager` / `team-completion-check` |
| Required Handoff Artifact | Legacy Completion Assessment reference only |
| Return Policy | メインエージェントへの返却は transport としてのみ許可し、workflow output として扱わない |
| Forbidden Outputs | user final response, main transport renderer, commit execution, quality approval without evaluation |

## Report / Queue Boundary

この Gate role は現行 workflow の queue consumer ではない。
新規 task で `gate-task-assessor` inbox、role-report、provider turn、tmux resident process を作ってはならない。
旧 Task Detail を読むときも、互換 section の意味を `teams-project-manager` / `team-completion-check` の現行 artifact へ写像するだけにする。

## Builder Precheck Compatibility

旧 `assessor-precheck` は互換 alias として残すが、正本 command は `team-completion-check` である。

| Precheck | Required |
|---|---|
| Command | `itb_bootstrap_builder.py team-completion-check` |
| Default phase | `post_routing` |
| Pass condition | `precheck_status: pass` |
| Block behavior | `validation_errors` を TPM への差し戻し理由にし、独立 assessor LLM 再監査へ進まない |
| Pass behavior | assessor runtime を起動せず、`completion_chain.yaml` に従って `gate-task-evaluator` へ進む |

## Current Flow Mapping

| Legacy Concept | Current Source Of Truth |
|---|---|
| old Completion Assessment | `Team Completion Check` thin section plus `team-completion-check` JSON artifact |
| old assessor status | `status` / `next_phase_allowed` in `team-completion-check` |
| old missing team evidence | `missing_evidence` / `blockers` in command artifact |
| old handoff to evaluator | `auto_queue_handoffs` entry `teams-project-manager -> gate-task-evaluator` |

## New Task Rules

新規 task では次を禁止する。

| Forbidden | Replacement |
|---|---|
| `gate-task-assessor` を resident / queue consumer として起動する | `team-completion-check` command |
| Team task board や review evidence を assessor LLM に再読込させる | TPM role-report と command artifact |
| assessor 名義の Markdown 判定表を Task Detail に追加する | `task-detail-append` による `Team Completion Check` thin section |
| evaluator への handoff を assessor が手書きする | builder / queue-watch の auto handoff |

## Legacy Read Compatibility

過去 task に旧 assessment section が残る場合は、次の互換情報として読む。

| Old Field | Interpret As |
|---|---|
| `ready_for_evaluation` | 現行 `team-completion-check.status: pass` 相当。ただし新規証跡ではない |
| `incomplete` / `blocked` | 現行 `blockers` / `missing_evidence` 相当 |
| old reasons | TPM または Director へ戻す修復 hint |

この互換読み取りは完了証跡を新規生成しない。
現行フローの complete 判定には、必ず `team-completion-check` command artifact と linked Task Detail thin section を使う。
