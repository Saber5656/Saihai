## 0. 先に:選んだ 7 案を「束」として設計する

7 案はバラバラに作らず、skydoves 式ポートフォリオとして設計すると総和が跳ね上がる。

- **共通コアライブラリを 1 つ作る。**
  - #1 Saihai / #4 blackbox / #6 tokenmeter は全部「各エージェント CLI の
    transcript・state を読む」処理が核 → **provider adapter 層を共通パッケージ化**。
  - #2 mcp-doctor / #3 skillet は「各クライアントの設定ファイル探索」が共通。
  - #8 agentci の記録リプレイは #4 blackbox の記録形式をそのまま使う。
  - つまり実体は「コア 2 つ + 薄いプロダクト 7 つ」。メンテコストが 1/3 になる。
- **GitHub Org を 1 つ立て、命名・ロゴ・README テンプレを統一**し、
  全 README の末尾に相互リンク(skydoves の相互送客)。
- **リリース順はテンポ重視:** 小さい #2 → #6 で毎月「新作」を出しつつ、
  #1 の国際化を進める。「毎月何か出す人」という認知がフォロワーを作る。

## A. ターミナル / CLI ツール

運用系 CLI・TUI は lazygit / k9s / thefuck が証明した永遠のスター鉱脈。
GIF 1 本で価値が伝わり、`brew install` 1 行で試せるのが強い。

### 1. undo — シェルに Ctrl+Z を
- **ピッチ:** `rm` / `git reset --hard` / 上書きリダイレクト等の破壊的操作を
  シェルフックで検知し、自動スナップショット。`undo` と打つだけで直前の事故を巻き戻す。
- **なぜバズる:** 「うっかり rm した」体験は全開発者共通のトラウマ。恐怖→安心の
  物語は Show HN で最も拡散しやすい型。エージェントに シェルを渡す時代になり
  「AI が変なコマンドを打っても戻せる」という 2026 年的な再文脈化ができる。
- **既存との差:** trash-cli(rm だけ)、rip(rm だけ)に対し「シェル操作全般の
  タイムライン+選択的リストア」。btrfs/APFS スナップショットがあれば利用、
  無ければファイルジャーナルにフォールバック。

### 2. whydid — 失敗コマンドの検死官
- **ピッチ:** コマンドが非ゼロ終了した瞬間に `whydid` と打つと、stderr・exit code・
  直近履歴から原因を平文で説明し、修正コマンドを提案(オフライン規則ベース+
  任意でローカル LLM)。
- **なぜバズる:** thefuck(タイポ修正、~90k★)が需要を証明済みで、その「エラー原因
  説明」への現代化はまだ定番不在。LLM なしでも動く設計が「また AI ラッパーか」
  という反発を避ける。
- **既存との差:** thefuck はタイポ訂正のみ。explainshell は構文解説のみ。
  「実行結果の診断」は空席。

### 3. logbook — なんでも SQL+チャート TUI
- **ピッチ:** `logbook access.log` / `logbook data/*.parquet` と叩くと、DuckDB が
  裏で立ち上がり、スキーマ推定済みのテーブルに SQL を打てて結果が即チャートになる TUI。
- **なぜバズる:** 「grep と jq の合わせ技に疲れた」は万人の痛み。DuckDB の波
  (組み込み分析 DB のデファクト化)にツール層で乗る。スクショ映え抜群。
- **既存との差:** harlequin(DuckDB TUI)は SQL エディタ寄り。「ログ・雑多ファイル
  への即席分析+可視化」に振り切る。lnav とは SQL 第一で差別化。

### 4. sshdeck — チームの SSH 管制卓
- **ピッチ:** ホスト・踏み台・ポートフォワードを YAML(git 共有可)で管理し、
  ファジー検索で接続、アクティブなトンネルを一覧表示する TUI。
- **なぜバズる:** Termius(有料 SaaS)の機能を「設定は git、通信はローカル」で
  置き換える。インフラ系は良いツールを見つけると社内布教する習性があり B2B 波及が速い。
- **既存との差:** sshs(単機能セレクタ)に対し、トンネル管理+チーム共有+
  known_hosts 監査まで含む「管制卓」。

### 5. cidebug — CI のタイムトラベルデバッガ
- **ピッチ:** 落ちた GitHub Actions の run ID を渡すと、同等コンテナをローカルに
  再構成し、失敗ステップの直前状態で止めてシェルに入れる。
- **なぜバズる:** 「CI でだけ落ちる」は全開発者の最悪の時間泥棒で、CI コスト高騰
  (分課金)への不満も追い風。act(41k★)が母数を証明しているが「再現の忠実さ」
  への不満が大量に残っている。
- **既存との差:** act は「ワークフローを手元で走らせる」。cidebug は「実際に落ちた
  run を法医学的に再現する」。難易度は高いが、当たれば act 級。

## B. AI × 開発自動化(エージェント運用基盤とは別領域)

「AI が苦役を肩代わりして PR を出す」型は、デモ動画(issue → PR が生える)が
そのまま宣伝になる。選んだ 7 案(運用基盤)と違い、こちらは**特定の苦役に特化した
垂直エージェント**。

### 6. i18n-agent — 翻訳苦役の自動化
- **ピッチ:** リポジトリを監視し、新規/変更された UI 文字列を検知 → 用語集・
  トーンガイドに従って翻訳 → ロケールファイルへ PR。差分だけの人間レビュー UI 付き。
- **なぜバズる:** i18n は「重要なのに全員が後回しにする」代表的苦役。
  「対応言語が 2 → 20 になった」というビフォーアフターが刺さる。
  グローバル OSS ほど恩恵が大きく、OSS メンテナ層(=スターを付ける層)に直接刺さる。
- **既存との差:** inlang / Crowdin はプラットフォーム(場)を提供。i18n-agent は
  「働き手」を提供する。lockfile 的な翻訳メモリを repo 内に置き、ベンダーロックなし。

### 7. docdrift — ドキュメント乖離の検知と修復
- **ピッチ:** CI で「コードと README/docs の食い違い」(消えたフラグ、変わった
  デフォルト値、古いサンプルコード)を検知し、修正 PR を提案。
- **なぜバズる:** 「ドキュメントは書いた瞬間から腐る」は開発の普遍的あるある。
  サンプルコードの実行検証(doctest 的)という決定的機能から入ると信頼が得やすい。
- **既存との差:** 既存はリンク切れチェッカー程度。「意味的な乖離」を対象にするのが新しい。

### 8. onboard — コードベースの自動ツアーガイド
- **ピッチ:** git 履歴・依存グラフ・AI 解析から「このリポジトリを理解する 10 ステップ
  ツアー」を生成し、エディタ/ブラウザで対話的に歩ける。新人の最初の 1 週間を 1 日に。
- **なぜバズる:** オンボーディングコストは全チームの見えない大出費。
  「新しく入った人がまず走らせるコマンド」という明確な使用シーンがある。
- **既存との差:** VS Code CodeTour は手書きツアーの再生機。onboard は生成+
  履歴に基づく「なぜこうなっているか」の考古学。

### 9. a11y-bot — アクセシビリティを直すボット
- **ピッチ:** axe 等の検査で違反を見つけるだけでなく、コンテキストを読んで
  「正しい alt テキスト・ラベル・フォーカス順」の修正 PR を出す。
- **なぜバズる:** 欧州アクセシビリティ法(2025 年 6 月施行)で対応が法的義務になり、
  企業が「探しに来る」カテゴリになった。社会的意義があるため報道・カンファレンスに
  取り上げられやすい(スター以外の増幅経路を持つ)。
- **既存との差:** 既存(axe, Lighthouse)は検出のみ。「修正まで」が空席。

### 10. reaper — 言語横断デッドコード刈り取り機
- **ピッチ:** JS/TS/Python/Go/... のデッドコード検出器を束ね、確信度スコア付きで
  「安全に消せる順」に並べ、1 クリックで削除 PR。
- **なぜバズる:** 「消すのが一番いいリファクタリング」は開発者の共通信仰。
  knip(JS 専用)の伸びが需要を証明。AI コーディングでコードが増殖する時代への
  カウンター(「増やす AI」に対する「減らす AI」)という物語が立つ。
- **既存との差:** 単言語ツールの上に「横断＋確信度＋自動 PR＋消した行数ダッシュボード」。

## C. Web 開発

### 11. stackblocks — shadcn 方式のフルスタック機能注入
- **ピッチ:** `stackblocks add auth` で、UI コンポーネントだけでなく API ルート・
  DB スキーマ・メール送信まで含む「機能の縦割り一式」が自分のコードとして
  リポジトリにコピーされる。依存ではなく所有。
- **なぜバズる:** shadcn/ui の「コピペ所有」モデルは今世代最強の OSS 流通形態と
  証明済みで、その適用範囲を UI → フルスタックに広げる枠はまだ王者不在。
  AI コーディングとの相性も抜群(エージェントに「blocks を使え」と指示できる)。
- **既存との差:** SaaS スターターキット(有料 boilerplate)を「必要な機能だけ・
  後から・自分の stack に」注入できる形に分解する。

### 12. dbbranch — ローカル DB のブランチング
- **ピッチ:** `dbbranch checkout feature-x` で、git ブランチに対応する Postgres/MySQL
  の copy-on-write クローンが即座に用意される。マイグレーション実験が怖くなくなる。
- **なぜバズる:** Neon が証明した「DB ブランチング」体験を、クラウド契約なしで
  ローカル Docker に持ち込む。「migration を試して壊して即戻す」デモが強い。
- **既存との差:** Neon/PlanetScale はクラウド。dbbranch はローカル開発専用に
  割り切ることで導入ゼロ設定を実現。

### 13. snapapi — API 時代の VCR
- **ピッチ:** 開発中の実 API トラフィックを記録し、型定義・モックサーバ・契約テストを
  自動生成。外部 API が落ちていてもテストが回る。
- **なぜバズる:** フレイキーな外部 API 依存テストは全チームの痛み。言語横断
  (プロキシ方式)にすることで Ruby の VCR 的資産を全生態系に開放する。
- **既存との差:** VCR/Polly.js は言語内ライブラリ。プロキシ+コード生成で言語非依存に。

### 14. mailblocks — メールテンプレの shadcn
- **ピッチ:** 認証メール・請求書・ダイジェスト等の実務メールテンプレ 50 種を、
  React 縛りなし(MJML/プレーン HTML)でプレビューしながらコピペ導入する CLI。
- **なぜバズる:** 「メールの HTML は 2026 年になっても地獄」という不変の痛み。
  小さく作れて、stackblocks(#11)への導線にもなる。
- **既存との差:** react-email は React 専用。フレームワーク非依存+日本語テンプレ
  (請求書・領収書など日本商習慣対応)を含めると国内でも独自価値。

## D. ローカル AI アプリ(消費者寄り)

GPU 不要級の小型モデル(音声認識・TTS・翻訳)が実用になった 2025-26 は、
「クラウド AI サービスのローカル版」を作る黄金期。プライバシー訴求は HN と相性最強。

### 15. kotodama — 日本語特化のローカル音声入力
- **ピッチ:** キーを押して話すと、その場で文字が入る системワイド音声入力。
  ただし日本語に全振り:口語→文語整形、フィラー(えー、あの)除去、
  「です・ます/だ・である」変換、IME 辞書連携、技術用語カスタム辞書。
- **なぜバズる:** 汎用ローカル音声入力は既に激戦(OpenWhispr、Whispering、Handy 等)
  だが、**どれも日本語の後処理が弱い**。「Whisper の生出力」と「日本語として
  出したい文」のギャップこそが価値。国内で圧勝→日本語圏の巨大な潜在ユーザー。
- **既存との差:** 競合の弱点(かな漢字の誤変換、句読点、敬語)に特化。
  Zenn/X 日本語圏で着火しやすく、あなたのホームグラウンド。

### 16. dubstudio — オープン吹き替えスタジオ
- **ピッチ:** 動画を放り込むと、文字起こし→翻訳→話者の声質を保った音声合成→
  字幕焼き込みまでのパイプラインが走る。YouTuber・教育動画・社内動画の多言語化。
- **なぜバズる:** ビフォーアフター動画がそのまま SNS コンテンツになる自己拡散型。
  クリエイター経済×AI×OSS の交点。
- **既存との差:** 既存(pyvideotrans 等)は中国語圏中心で UX が荒い。
  「品質レビュー UI(訳文・タイミングの人間修正)」を一級市民にして差別化。

### 17. cullai — 写真セレクトのローカル AI
- **ピッチ:** 撮影後の 3,000 枚から、ピント・目つぶり・構図・重複を判定して
  ベストショットを提案。RAW 対応、Lightroom への星評価書き戻し。
- **なぜバズる:** 写真家は「セレクトが一番苦痛」と全員言う。商用(Aftershoot、
  Narrative Select)が有料 SaaS で成立している=支払い意思のある痛み。
  OSS 空席。写真コミュニティは開発者コミュニティ外への波及口になる。
- **既存との差:** クラウドアップロード不要・買い切り不要。ワークフロー統合
  (XMP サイドカー)を丁寧に。

### 18. openkaraoke — カラオケ動画メーカー
- **ピッチ:** 音源からボーカルを除去(Demucs)し、歌詞をタイミング同期(Whisper
  アライメント)して、その場でカラオケ動画を生成。
- **なぜバズる:** 実用性より「愉快さ」で拡散する型。家庭・パーティ・配信で使われ、
  非開発者にリーチする。技術的には既存部品の組み合わせで数週間で作れる。
- **既存との差:** 部品(UVR、Demucs)は有名だが「カラオケ体験」までの統合が無い。

### 19. earmark — あとで読むを、朝のポッドキャストに
- **ピッチ:** 保存した記事を毎朝ローカル TTS(近年の高品質小型モデル)で
  10 分のダイジェスト音声にして podcast フィード配信。通勤で「あとで読む」が消化される。
- **なぜバズる:** Pocket 終了(2025)で read-later 難民が発生中、受け皿 OSS
  (Karakeep 等)は「保存」は解いたが「消化」は未解決。「積読が消える」は
  誰にでも刺さる約束。
- **既存との差:** 既存 read-later の後段に挿さるプラグイン設計にすれば、
  競合ではなく全 read-later ユーザーが対象になる。

## E. セルフホスト(生活インフラ)

Immich(open Google Photos)が示した通り、「毎月課金 SaaS の買い切り自宅版」は
現代 OSS で最もスターが集まるカテゴリ。開発者以外にも広がるのが特徴。

### 20. hearthboard — 家族の壁掛けハブ
- **ピッチ:** 古いタブレットを壁に貼ると、家族共有カレンダー・家事当番・献立・
  買い物リスト・写真スライドショーが出る。Google/iCloud カレンダー同期、子ども向け UI。
- **なぜバズる:** Skylight Calendar(約 $300+サブスク)が米国で爆売れ中で需要は
  証明済み。**OSS の定番は不在**(現状は Home Assistant で各自 DIY している状態)。
  「うちの壁に付けた」写真投稿が SNS で連鎖する製品体験。
- **既存との差:** HA の DIY 組み立てに対し「1 コンテナで完成品」。
  HA 統合(HACS プラグイン)を配布チャネルとして使う。

### 21. keepsake — デジタル終活キット
- **ピッチ:** 自分に万一があったとき、家族がアカウント・資産・重要書類に
  たどり着けるようにする:暗号化 Vault+デッドマンスイッチ(定期生存確認)+
  印刷可能な「家族向け手順書」生成。
- **なぜバズる:** 全人類に関係があり、感情に触れ、かつ既存解が「紙のノート」か
  「高額サービス」しかない。HN・Reddit で定期的に「どうしてる?」スレが立つ
  未解決問題。信頼が命の領域なので OSS(監査可能)であること自体が最大の売り。
- **既存との差:** パスワードマネージャの緊急アクセス機能は単機能。
  「手順書+スイッチ+Vault」の統合体験と、遺す側/受け取る側 両方の UX 設計。

### 22. fitvault — 運動データの自己主権
- **ピッチ:** Strava / Garmin / Apple Health / Fitbit から全履歴をインポートし、
  ローカルで生涯トレーニングログ・ヒートマップ・レコード分析を持つ。
- **なぜバズる:** Strava の API 制限強化(2024)以降「自分の走行データは自分のもの」
  という機運が高い。運動系開発者(多い)の心を直撃する。
- **既存との差:** 既存(FitTrackee 等)は記録アプリ。fitvault は「全サービスからの
  救出+分析」に主眼を置くデータハブ。

### 23. vitals — 健康データの統合ダッシュボード
- **ピッチ:** 体重計・睡眠・心拍・血圧・検診結果を 1 つのローカルタイムラインに。
  家族分の管理、医師に見せる PDF 出力。
- **なぜバズる:** ウェアラブル普及でデータは増えたのに、横断ビューはベンダー毎に
  分断。健康×プライバシーはセルフホスト動機が最も強い領域。
- **既存との差:** 単一デバイス系 OSS(Gadgetbridge)の上位レイヤーとして共存。
  #22 と部品共有可(取込アダプタ層)。

### 24. inkboard — 電子ペーパーダッシュボード FW
- **ピッチ:** 天気・カレンダー・ToDo・株価などのウィジェットをプラグインで組める
  e-ink ダッシュボード基盤。Raspberry Pi / 各社 e-ink 端末対応、省電力レンダリング。
- **なぜバズる:** e-ink ガジェットは HN/Reddit の永遠の人気者で、「作ってみた」
  記事が定期的にバズる。個別プロジェクトは乱立するが**共通基盤+プラグイン生態系**は
  空席。生態系を取るとコミュニティが自走する(改廃が賑わう構造)。
- **既存との差:** MagInkDash 等の単発プロジェクトに対し、プラグイン API・
  テーマ・デバイスプロファイルを持つ「基盤」。#20 hearthboard と部品共有可。

## F. データ・知識

### 25. flashvault — ノートが問題集になる
- **ピッチ:** Obsidian/Markdown ノートやハイライトから間隔反復カードを自動生成し、
  CLI・モバイル PWA で毎日 5 分復習。ノートを「書いたら覚える」資産に変える。
- **なぜバズる:** 「メモは取るのに身につかない」は知的労働者の共通の後ろめたさ。
  Anki は強いが「カードを作る」手間で 9 割が脱落する — そこだけを AI で解く。
- **既存との差:** Anki 補完(エクスポート対応)として入り、生成品質で勝負。

### 26. datadiff — データセットの git diff
- **ピッチ:** CSV/Parquet/DB テーブル 2 つを渡すと、行・列・分布レベルの
  「意味のある差分」を PR レビュー風 UI で表示。パイプライン変更の影響確認に。
- **なぜバズる:** データエンジニアリングのコードレビューには「このデータ、
  変更前後でどう変わった?」を見る標準ツールがまだ無い。CI に挟める
  (`datadiff --fail-on schema-change`)ことで日常導線に入る。
- **既存との差:** 既存(daff 等)は CSV 特化で古い。Parquet/DB 対応+統計的差分
  +CI ゲートで現代化。

## G. セキュリティ・プライバシー

### 27. permascan — アプリ権限の大掃除
- **ピッチ:** Google / GitHub / Slack / Microsoft アカウントに紐づく OAuth 許可・
  API トークン・アプリ連携を横断で棚卸しし、「1 年使ってない・権限過剰」を提示、
  一括失効を支援する CLI+ローカル UI。
- **なぜバズる:** 全員が「昔許可した謎のアプリ」を数十個抱えている。サプライチェーン
  経由の情報漏えい事故が起きるたびに需要が再燃する(ニュース連動型の伸び方をする)。
- **既存との差:** 各サービスの設定画面を個別に回る以外の既存解が無い。

### 28. optout — データブローカーからの削除請求自動化
- **ピッチ:** 個人情報を売買するデータブローカー各社への削除請求(CCPA/GDPR 準拠)を
  自動化・進捗管理する。Incogni(月額 SaaS)のセルフホスト版。
- **なぜバズる:** プライバシー系 SaaS の中でも「気持ち悪さ」への訴求が最強のカテゴリ。
  HN 民の琴線ど真ん中。
- **注意:** ブローカー側仕様変更への追従コストが高く、米国中心の題材。
  コミュニティでルール定義を回す設計(ルール=データ化)が前提。30 案中で最も
  「維持体制」が問われる案。

## H. 学習・遊び

### 29. buildyourown — 「車輪の再発明」教習所
- **ピッチ:** 「Redis を自作する」「git を自作する」「SQLite を自作する」を、
  段階的テストが導く形で進めるオープン教材ハーネス(テストランナー+課題+解説)。
  CodeCrafters(有料)のオープン版。
- **なぜバズる:** awesome な「build-your-own-x」リンク集が 30 万★ 級 —
  需要は OSS 史上級に証明済みなのに、「実際に手を動かす統一体験」の OSS は不在。
  教材はコミュニティ貢献(新コース追加)が自然に回る=「改廃が賑わう」構造の見本。
- **既存との差:** リンク集ではなく実行環境。言語自由(テストは I/O 契約で判定)。
  AI 時代に「基礎を理解したい」揺り戻しの波もある。

### 30. termrace — ターミナル対戦ゲーム
- **ピッチ:** `termrace join friend` でターミナル同士を繋ぎ、タイピング速度・
  シェル芸・vim 操作のリアルタイム対戦。リプレイ共有付き。
- **なぜバズる:** 開発者の遊び系は配信・カンファレンス・社内イベントで使われて
  勝手に露出する。実装は小さく、メンテも軽い。ポートフォリオの「愛され枠」。
- **既存との差:** 単発のタイピング TUI はあるが「対戦・観戦・リプレイ」の
  ソーシャル設計を持つものは無い。

## 30 案からの選び方(私見)

**イチ推し 5 つ**(独断。理由は「痛みの普遍性 × 空席度 × あなたの適性 × 拡散構造」):

1. **#1 undo** — 全開発者の恐怖に効く。AI エージェント時代の再文脈化
   (「エージェントの事故も戻せる」)で、選んだ 7 案の物語とも接続する。
2. **#6 i18n-agent** — 「垂直エージェント」枠の最良。OSS メンテナ自身に刺さるため
   スター転換率が高く、エージェント設計スキルがそのまま活きる。
3. **#20 hearthboard** — Immich 型「生活系セルフホスト」の空席。
   開発者コミュニティの外へ波及する唯一無二のポテンシャル。
4. **#29 buildyourown** — 「改廃が賑わう OSS」という当初の目標に最も合致。
   コース追加という貢献の形が明確で、コミュニティが自走する。
5. **#15 kotodama** — 日本語圏というホームで確実に勝てる差別化があり、
   日本語圏で着火→英語圏へ、という ccusage 型の勝ち筋をなぞれる。

**組み合わせの妙:** #22 fitvault と #23 vitals(取込アダプタ共有)、
#20 hearthboard と #24 inkboard(レンダリング基盤共有)、
#11 stackblocks と #14 mailblocks(ブロック流通基盤共有)のように、
2 個 1 セットで共通コアを持たせると skydoves 式の量産が効く。

**やらない判断も重要:** 汎用音声入力・ミーティング議事録・deploy ツール・
写真管理(Immich)・パスワード管理(Vaultwarden)は既に王者か激戦区のため、
本リストから意図的に外すか角度を変えてある(→ #15 は日本語特化に変形)。
