# 依頼: 小さくて楽しいデスクトップ OSS ツール 4 本の設計

## 背景

個人で OSS を複数リリースしていく取り組みの一環として、まず
「開発規模が小さい」「非開発者にもおもしろいと思ってもらえる」
デスクトップの遊び心系ツールを 4 本作ろうとしています。
まだアイデア段階で、詳細要件は固まっていません。各プロダクトについて、
コードを書く前の設計(スコープ・技術選定・アーキテクチャ・UI・配布方法など)
をお願いします。

対象は以下の 4 つです。技術的に重ならないので並行で進めて構いません。

## 1. cursorpets

マウスカーソルを追いかける小さなキャラクターをデスクトップに常駐させる、
往年の「oneko」(猫がカーソルを追いかけるジョークソフト)の現代版。
ノスタルジー狙いで、実装自体は非常に小さく作れる見込み。

## 2. notifybird

OS の通知を、鳥がくわえて画面上を飛んで届けるような演出でスキン化する
デスクトップ常駐ツール。通知の中身そのものを変えるのではなく、
「届き方」を可愛く演出するのが狙い。

## 3. keycat

Enter キーを押すと画面の下からネコのキャラクターがにゅっと出てきて、
Enter を叩くタイミングに合わせて一緒に手を叩いてくれるデスクトップ常駐アプリ。
普段は画面に隠れていて、キー入力のたびに登場する。連打するとコンボ演出、
コマンドの成否(exit code)に反応する、といった拡張も考えられる。
キー入力を監視する都合上、「特定のキーの押下タイミングだけを見て、
入力内容そのものは一切記録・送信しない」というプライバシー設計が
信頼性の要になる。配信(OBS など)で見栄えがすることも重視したい。

## 4. 8bitme

人物・家族写真をアップロードすると、ドット絵・レトロゲーム風のアバターに
変換してくれるツール。ローカルで完結させたい。

## 依頼したいこと

各プロダクトについて、以下のような観点を必要に応じて検討し、設計として
まとめてください。現時点で決まっていないことが多いので、妥当と判断した
内容で仮決めしてもらって構いません。

- v1(最初のリリース)に入れる機能と、入れない機能の線引き
- 対応プラットフォーム(macOS / Windows / Linux)とその優先順位
- 技術選定(言語・フレームワーク・主要ライブラリ)とその理由
- 主要コンポーネントと処理の流れ
- UI/UX(主要画面、設定項目、初回起動時の体験)
- キー入力や通知など OS のセンシティブな情報を扱うものについては、
  何を見て何を見ないかというプライバシー設計
- 配布方法(パッケージマネージャ経由か単体バイナリか、自動更新の要否)
- README の構成案(第一印象で魅力が伝わることを重視。バナー・デモ GIF・
  1 行でインストールできることを意識する)
- 実装前に検証しておきたいリスクや不確定要素

コードはまだ書かず、設計書の作成までをゴールにしてください。
また、最初から機能を詰め込みすぎず、小さく作って早く形にする方針を
守ってください。
