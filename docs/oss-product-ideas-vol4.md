# OSS プロダクト提案書 vol.4 — 採用傾向の学習に基づく 100 案

**前提:** これまでの採否判断(採用 76 / 不採用 64)から発案者の嗜好を抽出し、
その方向性に全振りして生成した追加候補。既出(採用済み・不採用とも)は再掲しない。
競合メモは 2026-07 時点の知識ベース。着手前に最新確認を推奨。名称はすべて仮。

---

## 0. 学習した嗜好プロファイル

### 採用された 8 つの型

| 型 | 根拠(採用例) |
|---|---|
| 1. エージェントの運用・計測・可視化 | vol.1 をほぼ全採用。Saihai を持つコア・アイデンティティ |
| 2. 「魔法の一撃」CLI — 巻き戻し・予行・自動特定 | undo / dryrun / culprit / timeback / cidebug。**安全**と**時間遡行**のテーマ |
| 3. 苦役を食って PR を出す垂直エージェント | i18n-agent / a11y-bot / reaper / freshshot 全採用 |
| 4. 取り込み・変換(X → 構造化データ) | scanmd を自ら発案。formfill / whatsthis / meishi / tablecat / glyphlab |
| 5. 家族・家庭の温かい道具 | hearthboard / famchat / kidquest / inkstory / keepsake / medicab / openintercom |
| 6. 遊び・演出・ゲーミフィケーション | tokenhp を自ら発案。partymode / pixelparty / regexdojo / openkaraoke |
| 7. 時間・記憶・歴史 | chronomap を自ら発案。chronicle / musicmap / worklog / quotable |
| 8. 日本文化×季節×工芸(お堅い系は除く) | nengajo / meishi / kotodama は採用、行政・規範系は全落選 |

### 不採用から学んだ「避ける型」

- 標準化・仕様策定(AOC)/ 受動的レポート・分析ダッシュボード(covermap, datacard, repohealth)
- 定量自己管理(fitvault, vitals)・家計(kakeibo)
- 事務・行政・文書規範(denshi-todoke, jimoto, keigo-lint, prosecheck)
- 「衛生」系セキュリティ(permascan, clearmeta, homeguard, certwatch)—
  ただし能動的に**守る・直す**もの(trustlock, restoredrill)は採用
- イベント運営事務(quizkit, hackathon-in-a-box)— 遊び本体(partymode)は採用

### 生成規則(この vol.4 の設計原理)

**「見せる」より「動く」。「管理する」より「守る・遊ぶ・残す」。「規範」より「魔法」。**
