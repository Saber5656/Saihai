# OSS プロダクト提案書 vol.5 — 「困難の正面突破」100 案

**フィードバックの反映:** vol.4 への指摘「AI にプロンプトを投げるだけでできるものは
OSS の価値をなさない。重要なのはドメインの解決」を生成規則に昇格させた。

## 0. vol.5 の選定基準(全案がこれを満たす)

1. **LLM 抜きでも成立する技術コアを持つ** — アルゴリズム・プロトコル・信号処理・
   計算幾何・制約ソルバ・フォーマット解析・計測校正・分散システム。
   AI は任意の飾りにしかならず、コアは決定論的に動く。
2. **ドメインの深いモデル化が参入障壁になる** — 「知識の結晶化」こそが OSS の資産。
   誰かが一度ちゃんと作れば世界中が再発明をやめられる種類の問題。
3. **OSS で未解決**(王者不在、または王者が老朽化・放置・有料 Windows 専用)。
4. 発案者の嗜好軸(救出・保存/家庭/ものづくり/音/遊び/日本文化)は維持する。

各表の「技術コア」列が「なぜプロンプトで代替できないか」の証明になっている。
競合メモは 2026-07 時点の知識ベース。名称はすべて仮。
